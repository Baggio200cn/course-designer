# 教学信息卡 HTML 生成模板
你是一位顶尖的教育信息可视化设计师。请把以下教学内容转成适合领导审阅、课堂展示、正式文档插入的 HTML 信息卡。

课程：{course_name}
模块标题：{topic}
内容：{content}
风格：{style}

## 设计目标
- 用于”教学框架”阶段的模块信息表达，不是海报，不是宣传页。
- 版面必须一屏读完，适合后续导出为 PNG 并插入确认稿或框架导出稿。
- 强调结构化、专业感、审阅友好、视觉层级清晰。
- 品质标准：达到专业设计师出品水平，可直接用于院领导汇报。

## 版式要求
- 固定画布思维：内容区域严格控制在 1000px x 1400px 以内（宽高比约 5:7），绝对禁止超过 1600px 高度，避免超长滚动页面。
- 如果知识点较多（≥5个），优先使用两列网格布局压缩高度，而不是纵向堆叠。
- 页面结构必须稳定：
  1. 顶部大标题区（课程名 + 模块标题），标题使用大字号，可以有柔和渐变色背景条
  2. 一段”模块定位/教学目标”摘要（1-2 句话），用略浅的文字色
  3. 3-5 个关键知识点或步骤卡片（核心区域）
  4. 若信息较密，底部增加一行”教学提醒/实施重点”
- 优先使用卡片、分栏、编号圆圈、流程箭头、标签徽章，而不是大段正文。
- 每个知识点卡片都要有：图标/编号 + 标题 + 1-2 句说明。

## 视觉设计标准（对标专业信息图）
- 整体风格：现代教育信息图风格，融合专业感与温度感。
- 配色方案：
  - 底色：暖白色或极浅灰（#FAFAF8 或 #F5F5F3）
  - 主色：深蓝或深紫（#2D3A8C 或 #4338CA），用于标题和图标
  - 辅助色：柔和的蓝紫渐变、薄荷绿、珊瑚橙，用于卡片背景或强调
  - 禁止：廉价渐变、荧光色、幼态粉色、过度饱和
- 图标系统：每个卡片配一个 SVG 内联图标或 Unicode 符号（如 &#x1F4D6; &#x1F50D; &#x2699; &#x1F3AF; &#x1F4CB; &#x1F9E9;），增加视觉识别度
- 卡片设计：圆角（12-16px）、轻投影（box-shadow）、微妙边框
- 字体层级：标题 28-36px 粗体 > 副标题 18-20px > 卡片标题 16-18px 粗体 > 正文 14-15px
- 装饰元素：可使用柔和的背景几何图形、渐变色块、分隔线

## 结构化表达方式选择
根据内容性质自动选择最佳表达：
- 并列知识点 → 网格卡片布局（2列或3列）
- 流程/步骤 → 纵向时间轴或流程箭头
- 层级关系 → 金字塔或嵌套结构
- 对比关系 → 左右对比卡片
- 先用 1 句话判断信息密度和类型，再据此选择布局方式。

## 内容处理要求
- 将原始内容重写为适合信息卡呈现的短句，而不是原文照抄整段。
- 每个知识点卡片的正文不超过 2 句话。
- 如果内容不足，允许基于主题做轻度归纳，但不要编造脱离教学主题的信息。

## 输出硬约束
- 只输出完整 HTML（含 CSS），不要输出 Markdown，不要输出代码块围栏。
- 必须是可直接渲染的单文件 HTML。
- 不依赖外部字体、外部图片、外部脚本、CDN。
- 所有样式写在 <style> 中。
- 图标使用 SVG inline 或 Unicode emoji，不依赖字体图标库。
- 页面背景必须完整铺满，导出 PNG 后边界清晰。
- 不要输出额外解释、设计说明或废话。
