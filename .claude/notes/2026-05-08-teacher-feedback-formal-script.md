# 老师反馈：正式讲稿质量从 55→62 分卡住，未能"真正到位"

> **日期**：2026-05-08（v3.1.0 试点首次老师反馈）
> **来源**：试点学校老师真实使用反馈
> **触发场景**：老师粘贴 Adobe InDesign / Photoshop / Canva / 李宁年报 等参考 URL，重新生成正式讲稿
> **状态**：🔥 已升级到开发动作（方案 B 实施中），非临时笔记
> **关联里程碑**：方案 B 实施期间（预计 1-1.5 天）

## 一、老师原始反馈摘录（非改写）

> **整体判断**：原稿 55 分，新版 62 分。
> 它比原稿更安全、更规整，但**专业性和素材融合度没有明显提升，甚至课堂讲述的生动性变弱了**。
> 它解决了"虚构品牌案例"的一部分问题，但**没有解决你真正关心的"让大模型基于真实课程素材做专业推理"的问题**。

### 进步项（3 条）

1. 删除原稿里不可靠的具体品牌案例（"波司登 2024 冬季企划案""UR 满 200 减 80""太平鸟 2024 小红书海报"等）
2. 课程结构更清楚了，拆成"任务 1、任务 2、任务 3、任务 4"，课堂管理动作更明确
3. "真实国产服装品牌夏季新品宣传海报"改成任务驱动，比直接指定李宁真实活动更稳妥

### 主要问题（6 条）

1. **参考链接没真正进入讲稿**——Adobe InDesign 的父页面 / 段落样式 / 对齐分布、Photoshop 的 Select Subject / 图层透明度、Canva 的 Brand Kit / 小红书尺寸规范、李宁年报数据，**这些都没被转化成具体教学内容**。讲稿仍然是泛泛的"新建文档、对齐对象、替换素材"。

2. **内容变得更空泛了**——"大家要认真学习""有任何问题可以随时问我""保持整体协调性"等通用话术。这些话没错，但不能提高正式稿质量。

3. **"真实可靠素材"的价值没体现**——讲稿没明确告诉老师：投影展示哪一类真实资料 / 学生观察什么 / 资料来自哪里 / 如何从资料中提炼排版规则。

4. **课时逻辑断裂**——开头说"今天这第一课时"，模块 2 又说"上节课我们学习了……今天这节课来实操"。如果是同一节 90 分钟课，就不应说"上节课"；如果是两课时，时间结构要拆开。

5. **实操任务仍有版权风险**——"真实国产服装品牌""冰丝凉感 99 元 2 件""LOGO 放右上角"如果没有真实授权素材，最好明确为"模拟品牌"或"以国产运动品牌视觉风格为参考的模拟项目"。

6. **评分标准太粗**——"符合四大排版原则"太笼统。应该拆成可观察指标：标题与产品图是否左/中轴对齐、核心卖点字号是否为辅助文字 2 倍以上、LOGO 面积是否在画面 5%-10%、无关模板元素是否删除。

### 老师给的"正确范例"

```
教师投影展示：Adobe Photoshop 官方 Select Subject 教程截图。
教师提问：为什么服装海报制作中，一键选择主体比手动裁剪更适合课堂初学者？
教师总结：服装产品图通常需要从背景中突出出来，Select Subject 可以快速完成主体选择，
        再配合图层蒙版、透明度和文字层级完成海报布局。
学生操作：导入 T 恤产品图，使用选择主体功能抠出产品，将产品图放大到画面约 60%，
        再添加核心卖点文字。
```

**关键诉求**：参考资料不能只是"背景知识"，要变成"展示素材、观察问题、教师讲解、学生操作、评价标准"。

### 哪里"漏了"

| 漏点 | 位置 | 影响 |
|-----|------|------|
| **review 没"素材融合深度"维度** | src/main/services/review.service.js#24-33 | AI 浅引用（如只提"InDesign"）就过审，**审核没拦** |
| **prompt 没"五段式教学结构"约束** | prompts/formal-system.md | AI 把素材当背景扫读，没转化为"投影/观察/讲解/操作/评价"五段动作 |
| **没"全局课时连贯性"注入** | src/main/script/formal-generator.js | 分模块合成时全局上下文丢失，导致同一节 90 分钟课说"上节课" |
| **prompt 没"真实品牌→模拟品牌"安全规则** | prompts/formal-system.md | 实操任务有版权风险 |
| **prompt 没要求"评价标准必须可观察可量化"** | prompts/formal-system.md | 评分标准笼统 |

### 真正的根因（一句话）

**生成 prompt 强约束（必须引用 2 处具体内容）已做，但审核机制（review.service.js）只查 5 项通用质量，没有"素材融合深度"维度——AI 只要做"字符匹配级"浅引用就能过审。**

要打破"55→62 分天花板"，必须**在审核层加深度融合维度 + 改 prompt 输出格式为五段式**。

## 三、解决方案：方案 B（已采纳）

### 阶段 1（4 小时）

1. **review.service.js 加第 6 维度"素材融合深度"**
   - 检查讲稿是否含素材里的**具体技术名词**（不只是软件名）
   - 检查每个素材链接是否至少被深度引用 1 次
   - 不及格 → 走 reviseAndRevise

2. **prompts/formal-system.md 加"五段式教学结构"硬约束**
   - 每个知识点必须包含 5 段：📺 投影展示什么 / 👀 学生观察什么 / 🎤 教师讲什么 / ✋ 学生操作 / ✓ 评价标准

3. **formal-generator.js 注入"全局课时连贯性"约束**
   - 总学时 ≤ 1.5 时 → prompt 加"禁止使用'上节课'等跨节课表述"
   - 总学时 ≥ 2 时 → 允许"上节课"

### 阶段 2（4-6 小时）

4. **重写 formal-generator 输出格式**为五段式 JSON schema → JS 渲染

### 不在方案 B 内（推到将来或案 C）

- UI 加素材锚点（让老师手动绑定素材到知识点）
- 版权安全规则强校验
- 评价标准 schema 自动校验
